## 🎯 Цель проекта
•	**Какую задачу решает приложение:**  
Приложение предоставляет интерактивный доступ к данным о химических элементах периодической таблицы Менделеева.

•	**Почему эта задача важна:**  
Химические элементы — основа материального мира. Удобный справочник необходим студентам, ученым и всем, кто интересуется наукой.

•	**Целевая аудитория:**  
Студенты, школьники, преподаватели химии, научные сотрудники.
________________________________________
## 📌 Требования к приложению
**Функциональные требования**
•	Загрузка актуальных данных о химических элементах из внешнего JSON-ресурса.
•	Отображение списка элементов с основной информацией (символ, название, атомный номер, масса).
•	Кэширование данных во внутреннем хранилище устройства.

**Нефункциональные требования**
•	**Производительность:** Плавная прокрутка списка элементов.
•	**Удобство использования:** Простой и понятный интерфейс.
•	**Надёжность:** Обработка ошибок сети и корректный парсинг данных.

## 🧠 Общая архитектура
•	**Выбранный подход:**  
Близкий к MVC (Model-View-Controller), где MainActivity выполняет роль контроллера.

•	**Почему выбран именно он:**  
Проект имеет простую структуру, что делает использование сложных паттернов (вроде MVVM) избыточным на текущем этапе.

•	**Основные компоненты системы:**  
- **Model:** Класс `Element`.
- **View:** `RecyclerView` в `MainActivity`.
- **Controller/Logic:** `MainActivity` и `ElementAdapter`.

## 🧩 Подробное описание классов
**📌 Класс: MainActivity**
- **Роль:** Главный экран приложения.
- **Ответственность:** Инициализация UI, управление жизненным циклом, запуск фоновых задач по загрузке данных, взаимодействие с адаптером.
- **Основные методы:**
  - `onCreate()` — настройка RecyclerView, инициализация потоков.
  - `loadPeriodicTableData()` — логика загрузки данных (из файла или сети).
  - `downloadPeriodicTable()` — загрузка JSON файла из интернета.
  - `parseElements()` — парсинг JSON строки в список объектов Element.

**📌 Класс: Element**
- **Роль:** Модель данных.
- **Ответственность:** Хранение свойств химического элемента (название, символ, масса и т.д.).

**📌 Класс: ElementAdapter**
- **Роль:** Связующее звено между данными и UI.
- **Ответственность:** Создание элементов списка и заполнение их данными из списка `Element`.
________________________________________
## 🔄 Диаграмма работы приложения
1. Запуск приложения.
2. `MainActivity` проверяет наличие файла `periodic_table.json` во внутреннем хранилище.
3. Если файла нет, запускается `downloadPeriodicTable()` в фоновом потоке.
4. После получения файла (или если он уже был), данные считываются и парсятся методом `parseElements()`.
5. Результаты передаются в `mainHandler`, который обновляет UI через `ElementAdapter`.

## 🎨 UI/UX анализ
•	**Почему интерфейс сделан именно так:** Выбран классический список (RecyclerView) для обеспечения быстрой навигации по большому количеству элементов.
•	**Какие принципы использованы:**
–	Простота: Нет лишних элементов.
–	Логичность: Группировка данных элемента (символ, имя, номер).
–	Доступность: Четкие шрифты и контрастные цвета.
________________________________________
## ⚙️ Работа с потоками
•	**Используются:**
- `ExecutorService` (SingleThreadExecutor) для сетевых и дисковых операций.
- `Handler` (MainLooper) для обновления UI.
•	**Почему выбран этот способ:** Это надежный и классический способ управления многопоточностью в Java-Android приложениях без использования сторонних библиотек.
•	**Как предотвращаются:**
–	зависания (ANR): Тяжелые операции вынесены в фоновый поток.
–	утечки памяти: Остановка ExecutorService в методе `onDestroy()`.
________________________________________
## 💾 Работа с данными
•	**Где хранятся данные:** Внутреннее хранилище приложения (`getFilesDir()`), файл `periodic_table.json`.
•	**Почему выбран этот способ:** Обеспечивает доступность данных без интернета после первой загрузки.
•	**Как обеспечивается:**
–	сохранность: Файл записывается в приватную область приложения.
–	корректность: Использование `JSONObject/JSONArray` для валидации структуры.
________________________________________
## 🌐 Работа с сетью
•	**Как выполняются запросы:** Используется стандартный `HttpURLConnection`.
•	**Обработка ошибок:** Блоки try-catch для перехвата IOException и JSONException.
•	**Что происходит при отсутствии интернета:** Если файл уже загружен, приложение работает в офлайн-режиме. Если нет — выводится Toast с сообщением об ошибке.
