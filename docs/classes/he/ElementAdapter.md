# תיאור המחלקה ElementAdapter

## 1. מידע כללי
*   **שם המחלקה:** `ElementAdapter`
*   **סוג:** Adapter (עבור RecyclerView)
*   **תפקיד:** מחלקה זו פועלת כמתרגמת. היא לוקחת רשימה של אובייקטי `Element` והופכת אותם לשורות ויזואליות שניתן להציג על המסך.
*   **אינטראקציה:** 
    *   עובדת עם `MainActivity` שמספקת את הנתונים.
    *   עובדת עם `Element` כדי לחלץ מידע.
    *   משתמשת ב-`element_item.xml` כדי להגדיר איך כל שורה נראית.

## 2. משתנים (שדות המחלקה)

| שם | סוג | מטרה |
| :--- | :--- | :--- |
| `elements` | `List<Element>` | הרשימה הפנימית של היסודות הכימיים שיוצגו. |

## 3. מתודות המחלקה

### שם המתודה: `onCreateViewHolder`
*   **מה המתודה עושה:** היא יוצרת את עיצוב ה-XML עבור שורה בודדת. זה כמו לבנות את ה"מסגרת" לתמונה.
*   **מתי נקראת:** באופן אוטומטי כשה-`RecyclerView` זקוק לשורה חדשה להצגה.

### שם המתודה: `onBindViewHolder`
*   **מה המתודה עושה:** כאן קורה ה"ציור" בפועל. היא לוקחת `Element` אחד מהרשימה לפי המיקום שלו ומכניסה את השם, הסמל והמסה שלו לשדות הטקסט.
*   **מתי נקראת:** בכל פעם ששורה הופכת לנראית על המסך.

## 4. הסבר במילים פשוטות
דמיינו שאתם **מלצרים** במסעדה.
*   **רשימת היסודות** היא ההזמנה מהמטבח.
*   **עיצוב ה-XML** הוא הצלחת הריקה.
*   **onBindViewHolder** הוא הפעולה שלכם כשאתם שמים את האוכל (הנתונים) על הצלחת.
*   **RecyclerView** הוא הלקוח שמחכה לאוכל שלו.
אתם (האדפטר) לוקחים את האוכל מהמטבח ומגישים אותו בצורה יפה ללקוח.
