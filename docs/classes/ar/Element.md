# وصف فئة Element

## 1. معلومات عامة
*   **اسم الفئة:** `Element`
*   **النوع:** فئة عادية (POJO / Model)
*   **الغرض:** تعمل هذه الفئة كقالب أو "حاوية" لعنصر كيميائي واحد (مثل الهيدروجين أو الأكسجين). وهي تخزن جميع خصائص العنصر.
*   **التفاعل:** 
    *   تنشئ `MainActivity` قائمة من هذه الكائنات بعد تحليل JSON.
    *   يستخدم `ElementAdapter` هذه الكائنات لملء حقول النص في واجهة المستخدم.

## 2. المتغيرات (حقول الفئة)

| الاسم | النوع | الغرض |
| :--- | :--- | :--- |
| `name` | `String` | الاسم الكامل للعنصر (مثلاً "Helium"). |
| `symbol` | `String` | الرمز القصير (مثلاً "He"). |
| `number` | `int` | العدد الذري. |
| `atomic_mass` | `double` | الكتلة الذرية. |
| `summary` | `String` | وصف مختصر أو حقائق عن العنصر. |

*كيفية استخدامها:* هذه المتغيرات خاصة (private). يتم الوصول إليها باستخدام طرق "Getters" و "Setters" لضمان سلامة البيانات.

## 3. المنطق العام
هذا هو "حامل البيانات". يمكنك التفكير فيه كبطاقة في خزانة ملفات. تمثل كل بطاقة عنصراً واحداً ولها حقول لاسمها ووزنها ورمزها.

## 4. شرح بكلمات بسيطة
فكر في هذه الفئة كـ **بطاقة عمل** لعنصر كيميائي.
تحتوي البطاقة على حقول محددة: "الاسم"، "الرمز"، "رقم الهاتف" (العدد الذري).
الفئة نفسها هي مجرد *تصميم* البطاقة. عندما يعمل التطبيق، فإنه ينشئ 118 بطاقة "فيزيائية" (كائنات) ويملأها بالمعلومات.
