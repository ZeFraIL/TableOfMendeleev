# Описание класса ElementAdapter

## 1. Общая информация
*   **Имя класса:** `ElementAdapter`
*   **Тип:** Adapter (Адаптер для RecyclerView)
*   **Назначение:** Этот класс работает как переводчик. Он берет список объектов `Element` и превращает их в визуальные строки (элементы списка), которые можно отобразить на экране.
*   **Взаимодействие:** 
    *   Работает с `MainActivity`, которая предоставляет данные.
    *   Работает с классом `Element` для извлечения информации.
    *   Использует файл `element_item.xml` для определения внешнего вида каждой строки.

## 2. Переменные (Поля класса)

| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `elements` | `List<Element>` | Внутренний список химических элементов для отображения. | Используется почти во всех методах для определения того, что показать. |

## 3. Методы класса

### Имя метода: `onCreateViewHolder`
*   **Тип:** `public`
*   **Возвращаемое значение:** `ElementViewHolder`
*   **Параметры:** `ViewGroup parent`, `int viewType`
*   **Что делает метод:** "Раздувает" (создает) макет XML для одной строки. Это как создание "рамки" для будущей картины.
*   **Когда вызывается:** Автоматически, когда `RecyclerView` нужна новая строка для отображения.

### Имя метода: `onBindViewHolder`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `ElementViewHolder holder`, `int position`
*   **Что делает метод:** Здесь происходит реальное "рисование". Метод берет один `Element` из списка по его позиции (`position`) и помещает его имя, символ и массу в текстовые поля `holder`.
*   **Когда вызывается:** Каждый раз, когда строка становится видимой на экране при прокрутке.

### Имя метода: `updateElements`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `List<Element> newElements`
*   **Что делает метод:** Заменяет старый список на новый и вызывает `notifyDataSetChanged()`, чтобы экран обновился.
*   **Когда вызывается:** Когда `MainActivity` заканчивает загрузку данных.

## 4. Жизненный цикл
Адаптер не имеет стандартного жизненного цикла Android, но он тесно связан с `RecyclerView`.

## 5. Взаимодействие с интерфейсом (UI)
*   **`ElementViewHolder`**: Внутренний класс, который "держит" ссылки на элементы интерфейса (TextView) одной строки. Это ускоряет работу: при прокрутке приложению не нужно каждый раз искать элементы через `findViewById`.

## 6. Общая логика
1. Создать дизайн строки (`onCreateViewHolder`).
2. Заполнить строку данными (`onBindViewHolder`).
3. Подсчитать количество строк (`getItemCount`).

## 7. Простое объяснение
Представь, что ты **официант** в ресторане.
*   **Список элементов** — это заказ из кухни.
*   **Макет XML** — это пустая тарелка.
*   **onBindViewHolder** — это процесс выкладывания еды (данных) на тарелку.
*   **RecyclerView** — это клиент, который ждет еду.
Ты (Адаптер) берешь еду из кухни и красиво подаешь её клиенту.
