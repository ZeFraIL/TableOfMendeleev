# Описание класса Element

## 1. Общая информация
*   **Имя класса:** `Element`
*   **Тип:** Обычный класс (POJO / Модель)
*   **Назначение:** Этот класс служит шаблоном или "контейнером" для одного химического элемента (например, водорода или кислорода). Он хранит все характеристики элемента.
*   **Взаимодействие:** 
    *   `MainActivity` создает список этих объектов после обработки JSON.
    *   `ElementAdapter` использует эти объекты для заполнения текстовых полей в интерфейсе.

## 2. Переменные (Поля класса)
*В этом классе много полей. Вот основные:*

| Имя | Тип | Цель |
| :--- | :--- | :--- |
| `name` | `String` | Полное название элемента (например, "Helium"). |
| `symbol` | `String` | Короткий символ (например, "He"). |
| `number` | `int` | Атомный номер (порядковый номер). |
| `atomic_mass` | `double` | Атомная масса. |
| `summary` | `String` | Краткое описание или факты об элементе. |
| `shells` | `ArrayList<Integer>` | Конфигурация электронных оболочек. |

*Как это используется:* Эти переменные приватные (скрытые). Доступ к ним осуществляется через специальные методы "Геттеры" (get) и "Сеттеры" (set) для обеспечения безопасности данных.

## 3. Методы класса
Класс состоит в основном из **Геттеров** и **Сеттеров**.

### Имя метода: `getName` / `setName`
*   **Тип:** `public`
*   **Возвращаемое значение:** `String` (для get) / `void` (для set)
*   **Параметры:** `String name` (для set)
*   **Что делает метод:** `getName` возвращает имя, `setName` обновляет его.
*   **Когда вызывается:** Во время обработки JSON (`setName`) и при обновлении интерфейса (`getName`).

## 4. Жизненный цикл
Это обычный Java-класс, поэтому у него нет жизненного цикла Android. Он существует в памяти до тех пор, пока на него есть ссылка в списке или переменной.

## 5. Взаимодействие с интерфейсом (UI)
Напрямую не взаимодействует. Поля считываются адаптером (`Adapter`), который затем помещает их в элементы UI.

## 6. Общая логика
Это "носитель данных". Представьте его как карточку в картотеке. Каждая карточка представляет один элемент и имеет поля для его названия, веса и символа.

## 7. Простое объяснение
Представь, что этот класс — это **визитная карточка** химического элемента.
На карточке есть определенные поля: "Имя", "Символ", "Телефон" (Атомный номер).
Сам класс — это просто *дизайн* карточки. Когда приложение запускается, оно создает 118 "физических" карточек (объектов) и заполняет их информацией.
