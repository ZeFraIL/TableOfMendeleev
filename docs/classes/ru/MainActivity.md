# Описание класса MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность / Экран)
*   **Назначение:** Это главный экран приложения. Он отвечает за пользовательский интерфейс, управление жизненным циклом приложения и логику загрузки данных о химических элементах. Он решает, скачивать ли данные из интернета или читать их из памяти телефона.
*   **Взаимодействие:** 
    *   Использует `ElementAdapter` для отображения данных в списке.
    *   Использует класс `Element` для хранения обработанных данных.
    *   Взаимодействует с системой Android для выполнения сетевых запросов и операций с файлами.

## 2. Переменные (Поля класса)

| Имя | Тип | Цель | Где используется |
| :--- | :--- | :--- | :--- |
| `recyclerView` | `RecyclerView` | Специальный компонент UI для эффективного отображения длинных списков. | Используется в `onCreate` для настройки списка. |
| `adapter` | `ElementAdapter` | "Мост", который берет данные из списка и помещает их в `RecyclerView`. | Используется во всем классе для обновления интерфейса. |
| `executorService` | `ExecutorService` | Инструмент для запуска задач в "фоне" (не на главном потоке экрана). | В `loadPeriodicTableData` для работы с сетью/файлами. |
| `mainHandler` | `Handler` | Инструмент для отправки сообщений обратно в "Главный поток" (UI поток). | Для обновления списка после загрузки данных. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Сохраненные данные предыдущей сессии (если приложение было перезапущено). |
*   **Что делает метод:**
    1. Вызывает `super.onCreate` для базовой настройки Activity.
    2. Устанавливает макет из файла `activity_main.xml`.
    3. Находит `RecyclerView` в макете по ID.
    4. Создает `adapter` и подключает его к `RecyclerView`.
    5. Инициализирует `executorService` и `mainHandler`.
    6. Запускает процесс загрузки данных через `loadPeriodicTableData()`.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Что важно понимать:** Это точка входа. Если выполнять здесь слишком тяжелую работу, приложение "зависнет".

### Имя метода: `loadPeriodicTableData`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает метод:**
    1. Переключается на фоновый поток через `executorService`.
    2. Проверяет, существует ли файл `periodic_table.json` в папке приложения.
    3. Если файла нет, вызывает `downloadPeriodicTable()`.
    4. Читает содержимое файла через `readFile()`.
    5. Превращает текст в список объектов `Element` через `parseElements()`.
    6. Через `mainHandler` дает команду адаптеру показать новый список.
*   **Когда вызывается:** Вручную из метода `onCreate`.
*   **Что важно понимать:** Этот метод управляет основным потоком данных.

### Имя метода: `downloadPeriodicTable`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `file` | `File` | Путь, куда будут сохранены скачанные данные. |
*   **Что делает метод:**
    1. Открывает соединение с URL-адресом на GitHub.
    2. Читает данные из интернета небольшими порциями (буферами).
    3. Записывает эти порции в локальный файл на устройстве.
*   **Когда вызывается:** Из `loadPeriodicTableData`, если файл отсутствует.

## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается при первом создании Activity. Здесь строится интерфейс и запускаются службы.
*   **`onDestroy()`**: Вызывается прямо перед уничтожением Activity. Здесь мы останавливаем `executorService`, чтобы предотвратить утечки памяти (чтобы поток не продолжал работать после закрытия приложения).

## 5. Взаимодействие с интерфейсом (UI)
*   **`RecyclerView`**: Показывает список элементов. Связан через `findViewById(R.id.recyclerView)`.
*   **`Toast`**: Всплывающие уведомления для показа ошибок.

## 6. Взаимодействие с другими компонентами
*   **API/Сеть**: Подключается к GitHub для получения сырых JSON-данных.
*   **Файловая система**: Сохраняет JSON в `getFilesDir()` для работы без интернета.

## 7. Общая логика
Старт -> Проверка локального файла -> (Опционально) Скачивание -> Чтение текста -> Парсинг JSON -> Показ в списке.

## 8. Простое объяснение
Представь, что ты в библиотеке.
*   **MainActivity** — это библиотекарь.
*   **RecyclerView** — это книжная полка.
*   **Интернет** — это склад книг.
Библиотекарь проверяет, есть ли книга (данные) на полке. Если нет, он заказывает её со склада. Как только книга приходит, он расставляет информацию по местам на полке, чтобы ты мог её увидеть.

---
**Заметка по улучшению:** Код `recyclerView.setOnClickListener` в методе `onCreate` не будет работать как ожидается, так как отдельные элементы списка обрабатывают клики внутри адаптера. Рекомендуется перенести логику кликов в `ElementAdapter`.
